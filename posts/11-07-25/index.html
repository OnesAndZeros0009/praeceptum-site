Started Development

July 14, 2025

I started development!#

I did a few thing such as:

  • Drew some more “concept” art.
  • Created a window class for handling the window, who would have guessed? window.pollEvents(); window.clear(); window.update(); window.isOpen();
  • Implimented a chunk system to reduce memory usage and alow multithreading.

In order to save memory, chunks containing tiles with the same data are simplifyed to a “UniformChunk”.

Tile data is stored like so:

struct MaterialFraction {
    uint8_t material_id = 0;
    uint8_t mass = 0;
};

struct Tile {
    MaterialFraction materials[MAX_MATERIALS_COUNT];
    uint8_t materialsLength = 0;
    uint16_t temperature = 0; // measured in K (Kelvin)
    uint8_t totalMass = 0; // measured in KG (kilograms)
    uint8_t electricalConductivity = 0; // arbitrary measurement (0 - 100)
    uint8_t thermalConductivity = 0; // arbitrary measurement (0 - 100)

I use #pragma pack(push, 1) before the structure defenitions and #pragma pack(pop) after to eleminate any padding between objects. This causes a minor performence decrease due to misaligned byte however the difference is negligable.

Memory usage per tile with and without padding:

  • Without padding: 26 bytes
  • With padding: 28 bytes
